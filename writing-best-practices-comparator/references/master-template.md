# Per-Venue Master Note Template

The output of **Mode A (within-venue)** is `./Knowledge/{VenueFolder}/Writing-Best-Practices.md`. Use the structure below. The four-backtick fence around the template is so three-backtick fences inside it render correctly.

(For **Mode B (cross-venue)**, see `cross-venue-template.md` instead.)

## Structure overview

1. **Frontmatter** (Obsidian properties)
2. **Inventory** — table of analysed papers
3. **Manual section** (preserved across re-runs via `<!-- MANUAL-START -->` markers)
4. **Cross-paper comparison table** — moves × papers
5. **Universal rules** — recurring patterns (≥ 2 papers)
6. **Genre-specific rules** — patterns grouped by paper genre
7. **Genre coverage** — which genres are represented, which are gaps
8. **Linked notes**

## Notes on the template

1. **Manual block.** The `<!-- MANUAL-START -->` / `<!-- MANUAL-END -->` markers are sacrosanct. Everything between them is preserved across regenerations. Place them once in section 2 and don't move them.

2. **Universal rule count.** Aim for **5-10** universal rules. Fewer than 5 means the universality bar is too strict (or the corpus is too small); more than 10 dilutes "universal" into "common-ish."

3. **Genre coverage section.** Explicitly listing empty genres (no analyses yet for theory papers, position papers, etc.) is *not* a confession — it's a roadmap. It tells the user which papers to analyse next to round out the corpus.

4. **Citation format.** Use Obsidian heading-links to cite specific per-paper rules: `[[Writing-Best-Practices-Jiang-2025#Rule 5]]`. This makes the master genuinely navigable.

5. **When to regenerate.** Any time a per-paper note is added or substantially edited. The skill can be re-invoked safely — it preserves the manual block and creates a `.bak.md` of the old master.
