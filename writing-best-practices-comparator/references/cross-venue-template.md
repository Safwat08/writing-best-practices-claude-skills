# Writing Best Practices — Cross-Venue Master

> [!abstract] What this note is
> A cross-venue synthesis comparing the per-venue masters of {K} venue folders ({Venue 1}, {Venue 2}, …). It separates **venue-agnostic best practices** — rules that appear as universal in *every* compared venue, the safest rules to apply to any paper — from **venue-divergent tendencies** — moves prominent in one venue's corpus but not another's. Regenerated by `writing-best-practices-comparator` (Mode B); the manual block is preserved.

> [!warning] Read the confound caveat first
> With a modest corpus, most apparent "venue differences" are really **genre-mix differences** — e.g. if the NeurIPS folder contains theory papers and the ICML folder does not, a "theory-paper move" will look like a "NeurIPS move." Every claim in §5 (Venue-divergent tendencies) is annotated with whether the genre distributions are comparable. Treat unconfounded divergences as real; treat confounded ones as corpus artifacts.

## 3. Venue-agnostic best practices

Rules that appear as a **universal rule in every compared venue's per-venue master**. These are the highest-confidence, most transferable rules — they survive across conferences and are the core playbook for *any* paper regardless of target venue.

> [!success] Apply these to every paper, any venue
> 1. **{Rule.}** {one-line elaboration.} Universal in: {Venue1} ({n}/{N1} papers), {Venue2} ({n}/{N2}), … Cited per-venue: [[{Venue1}/Writing-Best-Practices#4. Universal rules\|{Venue1} #x]], [[{Venue2}/Writing-Best-Practices#4. Universal rules\|{Venue2} #y]].
> 2. **{Rule.}** {…}
> … (typically 6-12 rules — the intersection of the venues' universal-rule sets)

If the venue-agnostic set is small (< 5 rules), say so explicitly and explain: either the venues genuinely differ, or (more likely with a small corpus) the per-venue universal-rule sets were extracted with different granularity. Recommend the user grow each venue folder to ≥ 6 papers.

## 5. Venue-divergent tendencies

Moves that are prominent in one venue's corpus but absent / rare in another. **Each entry is annotated for the genre-mix confound.**

> [!example] Divergence 1 — {short name}
> **Observed:** {Venue A} papers do {X}; {Venue B} papers do {Y} (or omit X).
> **Evidence:** {n}/{NA} {Venue A} papers vs. {m}/{NB} {Venue B} papers.
> **Confound check:** {Are the genre distributions comparable? If {Venue A} is theory-heavy and {Venue B} is not, and X is a theory-genre move, this is a *genre artifact, not a venue effect* — say so. If genre mixes are comparable, this is a *candidate real venue tendency*.}
> **Verdict:** {⚠️ confounded by genre mix — not a reliable venue signal} OR {✓ unconfounded — candidate venue tendency, still tentative at this corpus size}.

{Repeat for each divergence. Order: unconfounded divergences first, confounded ones after.}

> [!info] If there are no unconfounded divergences
> State plainly: *"At the current corpus size, every observed cross-venue difference is explained by genre-mix differences between the folders. No venue-specific writing tendency can be claimed. To separate venue from genre, grow each folder so the genre distributions overlap."* An honest null result is more useful than an overclaimed venue effect.

## 7. How to use this note

- **Writing any paper:** apply every rule in §3 (venue-agnostic). These are venue-independent.
- **Writing for a specific venue:** start with §3, then open that venue's per-venue master ([[{Venue1}/Writing-Best-Practices]] etc.) for the genre-specific playbook. Consult §5 only for *unconfounded* divergences.
- **Deciding where to submit:** §5 is *not* a submission-strategy guide at this corpus size — the divergences are too confounded. Do not use it to pick a venue.

## Notes on the template

1. **§3 is an intersection, §5 is a difference.** §3 = rules universal in *all* venues. §5 = moves prominent in some venues, not others. Keep them distinct — conflating them is the most common failure mode.
2. **The confound caveat is mandatory.** Never present a §5 divergence without the genre-mix confound check. With < ~10 papers per venue, most divergences are confounded. An honest "no reliable venue effect" beats a confident wrong claim.
3. **Wikilinks to per-venue masters use the folder path** — `[[NeurIPS/Writing-Best-Practices]]` — because multiple venue masters share the basename `Writing-Best-Practices`. A bare `[[Writing-Best-Practices]]` would be ambiguous. Use the `folder/basename` form, optionally with a `|display` alias.
4. **§5 ordering:** unconfounded divergences first (the trustworthy content), confounded ones after (labelled as artifacts).
5. The cross-venue master is a *consumer* of per-venue masters. If a per-venue master is stale, the cross-venue master inherits the staleness — note it in §1's "Master last regenerated" column.
